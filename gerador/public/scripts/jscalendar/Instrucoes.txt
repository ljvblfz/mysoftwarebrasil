Utilização do DHTML Calendar.

O DHTML Calendar é uma solução em JavaScript para display de um calendário para seleção de datas.
Para sua utlização basta utilizar as TAGs do Smarty "dhtml_calendar_init" para inicialização do calendário na página e a TAG "dhtml_calendar" para associação do calendário a um campo da página.

Ex:
/*Cabeçalho da página*/

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
<head>
<title>{{$tituloPagina|default:"Título não informado"}}</title>
<link href="{{$RELATIVE_APP_ROOT}}/css/safci.global.css" rel="stylesheet" type="text/css" media="screen" title="SAFCI Global Templates" />
<link href="css/tema.css" rel="stylesheet" type="text/css" media="screen" />
</head>
<body>
<---Chamada para inicializar o calendário.--->
{{dhtml_calendar_init src="../libs/js/jscalendar/calendar.js" setup_src="../libs/js/jscalendar/calendar-setup.js" lang="../libs/js/jscalendar/lang/calendar-br.js" css="../libs/js/jscalendar/calendar-blue.css"}}
<div id="calendar-container"></div> <----Esta linha é necessária como indicador ao javascript do container do calendário.
................
................
<----- Campo para o qual a data será retornada, devemos adicionar um gatilho (botão, imagem...) para chamar o calendário. Observe que esse gatilho deve ser identificado com um "id" --->
<input type="text" name="data" id="data" value="{{$data}}" maxlength="10" size="10" />
<div class="botaoAuxilio">
<img src="../libs/js/jscalendar/calendar-blue.jpg" id="escolhe_data" title="Selecione a data"/> 
</div>
................
................
<--- Fim da página, após o formulário, acrescentamos a chamada para o calendário indicando o gatilho e o campo para o qual será retornado--->
{{dhtml_calendar inputField="data" button="escolhe_data" singleClick=true}}
...............
...............
</body>
</html>

As opções da TAG dhtml_calendar são:

inputField string null
    A identificação do campo do formulário.

displayArea string null
    Este é o ID de um <span>, <div>, ou qualquer outro elemento que gostaria de usar
    para mostrar a data corrente. Isto normalmente é útil somente se o campo da data é
    do tipo "hidden", como uma área para mostrar a data.

button string null
    O ID do gatilho do calendário. Este é um elemento (normalmente um botão ou uma imagem)
    que irá disparar um certo evento (normalmente um "click") para a função que cria e mostra o calendário.

eventName string “click”
    O nome do evento que irá chamar o calendário. O nome deve ser sem o prefixo "on",
    tal como "click" ao contrário de "onclick". Pode se usar opções do tipo focus, over, etc..

ifFormat string “%Y/%m/%d”
    O formato de retorno da data para o campo. Este formato será utilizado mesmo se o campo for do tipo "hidden".

daFormat string “%Y/%m/%d”
    Formato da data a ser mostrado na área de display do calendário (Se definido).

singleClick boolean true
    O calendário pode ser do tipo "single-click mode" ou "double-click mode". Se "true" (padrão) o calendário será do tipo
    "single-click mode".

disableFunc function null
    A function that receives a JS Date object. It should return true if that
    date has to be disabled, false otherwise. DEPRECATED (see below).

dateStatusFunc function null
    Uma função que recebe um objeto JS de data e retorna um boolean ou uma string.
    Está função permite ao desenvolvedor apontar uma certa classe CSS para alguma data, 
    podendo então mudar a aparência. Se a função retornar true então a data será desativada.
    Se retornar false nada especial acontece. Se retornar um string este será tomado como 
    uma classe CSS e adicionado ao elemento da data. Se o string retornado for 
    "disabled" então a data també será desativada.For more information please also refer
    to section 4.3.8.

firstDay integer 0
    Especifica qual dia deve ser mostrado como o primeiro dia da semana. Valores 
    possíveis são de 0 a 6; 0 significa Domingo, 1 Segunda, ..., 6 Sábado.
    O usuário final pode facilmente mudar isto também, bastando clicar no nome do
    dia no cabeçalho do calendário.

weekNumbers boolean true
    Se verdadeiro então o calendário mostrará o número de semanas.

align string “Bl”
    Alinhamento do calendário, relativo ao elemento de referência. A referência
    ao elemento é dinamicamente escolhida: Se um displayArea é especificado então
    este será o elemento de referência de outra maneira o campo de entrada da 
    data será o elemento de referência. For the meaning of the alignment characters please
    section 4.3.11.

range array [1900, 2999]
    Um array tendo exatemente 2 elementos, inteiros. O primeiro elemento é o ano
    mínimo disponível, o segundo elemento é o ano máximo que o calendário irá
    permitir.

flat string null
    Se desejar um calendário flat, passe o ID objeto pai.

flatCallback function null
    Pode-se disponibilizar essa função se o calendário é do tipo flat. Esta será
    chamada quando a data no calendário é mudada com uma referência ao objeto
    calendário. See section 2.2 for an example of how to setup a flat calendar.

onSelect function null
    Se uma função for definida então você terá que gerenciar o evento "click-on-date"
    por si mesmo. Leia no calendar-setup.js e tome como um exemplo o OnSelect handler
    que pode ser encontrado lá.

onClose function null
    Esta função será chamada quando o calendário precisa ser fechado. 
    You don’t need to provide one, but if you do it’s your responsibility 
    to hide/destroy the calendar. You’re on your own. 
    Check the calendar-setup.js file for an example.

onUpdate function null
    If you supply a function handler here, it will be called right after the
    target field is updated with a new date. You can use this to chain 2
    calendars, for instance to setup a default date in the second just after a
    date was selected in the first.

date date null
    Este permite definir qual será a data innicial onde o calendário estará
    posicionado. Se omitido então o calendário abrirá na data atual.y date.

showsTime boolean false
    Se true então o calendário irá permitir a seleção de hora.

timeFormat string “24”
    Pode ser "12" ou "24". "12" igual a "am\pm" e "24" igual a 24hs.

electric boolean true
    Marque este para false se deseja que o calendário mude a data no campo somente
    quando fechado.

position array null
    Especifica a posição [x, y], relativa ao canto superior esquerdo da página.

cache boolean false
    Marque para true se desejar enviar para a cache o objeto calendário. Isto 
    significa que um único calendário será usado para todos os campos que requererem
    um calendário popup.

showOthers boolean false
    Se true então os dias referentes a meses sobrepostos serão mostrados na visualização
    corrente, mas em uma cor diferente.
